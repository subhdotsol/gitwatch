## üöÄ Overview

**GitWatch** is an intelligent Telegram bot that transforms how you interact with GitHub repositories. Whether you're a maintainer managing multiple projects, a contributor hunting for good first issues, or a team coordinating development work, GitWatch brings the power of GitHub directly into your Telegram conversations.

**No more context switching. No more missed notifications. Just seamless GitHub collaboration from Telegram.**

### ‚ú® Key Highlights

- üéØ **Real-time Notifications** - Webhook-driven updates, no polling delays
- üß† **Smart Filtering** - Role-based notifications that understand what matters to you
- üí¨ **Two-way Integration** - Comment, assign, label, and close issues directly from Telegram
- üë• **Team-Ready** - Bind repositories to group chats for collaborative workflows
- üîê **Secure** - OAuth per user, fine-grained permissions, encrypted token storage
- ü§ñ **AI-Powered** - Auto-summarization, priority scoring, and intelligent insights

## üöÄ Getting Started

### Prerequisites

- Node.js 20+
- npm or pnpm or yarn
- Telegram Bot Token (from [@BotFather](https://t.me/botfather))
- GitHub OAuth App credentials
- PostgreSQL database (for production)

### Installation

```bash
# Clone the repository
git clone https://github.com/yourusername/gitwatch.git
cd gitwatch

# Install dependencies
npm install

# Set up environment variables
cp .env.example .env.local
# Edit .env.local with your credentials

# Run development server
npm run dev
```

### Environment Variables

Create a `.env.local` file with the following:

```env
# Telegram Bot
TELEGRAM_BOT_TOKEN=your_telegram_bot_token

# GitHub OAuth
GITHUB_CLIENT_ID=your_github_oauth_client_id
GITHUB_CLIENT_SECRET=your_github_oauth_client_secret
GITHUB_WEBHOOK_SECRET=your_webhook_secret

# Database
DATABASE_URL=postgresql://user:password@localhost:5432/gitwatch

# Optional: Redis for queue
REDIS_URL=redis://localhost:6379

# Next.js
NEXTAUTH_SECRET=your_nextauth_secret
NEXTAUTH_URL=http://localhost:3000
```

## üé® Product Philosophy

### Act-from-Telegram

GitWatch is built on the philosophy that you should be able to perform most GitHub actions without leaving Telegram. No context switching, no opening browsers, no interrupting your flow.

### Role-Based Intelligence

Not all notifications are equal. Maintainers care about security issues and high-impact bugs. Contributors want good first issues and unassigned tasks. GitWatch adapts to your role and filters noise automatically.

### Team-First Collaboration

Bind repositories to Telegram groups and enable your entire team to discuss, assign, and resolve issues collaboratively - all within the chat they're already using.
